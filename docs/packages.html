Overview

Nova has a simple, Git-based package system. Packages are GitHub repositories that contain Nova source files and a nova.json manifest. There is no central registry — any GitHub repository with a valid manifest can be installed as a Nova package.

Key concepts:

  • nova.json — a manifest file describing the package (name, version, entry point)
  • nova_packages/ — the local directory where installed packages are stored
  • nova install — the command to install a package from GitHub
  • nova test — the built-in test runner for packages and projects

Installing Packages

Install a package by pointing nova install at a GitHub URL:

nova install https://github.com/username/nova-http

This clones the repository into your project's nova_packages/ directory:

my-project/
  main.nova
  nova_packages/
    nova-http/
      nova.json
      http.nova
      ...

Note: The nova_packages/ directory is created automatically if it does not exist. You should add it to your .gitignore if you are using version control.

Installing Specific Versions

Package versioning follows Git tags. By default, the latest commit on the default branch is installed.

Using Packages

Once installed, import the package by its name (as declared in its nova.json):

import "nova-http"

let response = nova_http.get("https://api.example.com/data")
print(response.body)

Nova resolves imports in the following order:

  1. Built-in standard library modules (math, io, etc.)
  2. Files relative to the current script
  3. Packages in nova_packages/

Creating a Package

A Nova package is simply a directory (and GitHub repository) containing:

  1. A nova.json manifest file
  2. One or more .nova source files
  3. Optionally, test files (*_test.nova)

Step 1: Create the Directory

mkdir nova-mylib
cd nova-mylib

Step 2: Write the Manifest

Create a nova.json file at the root of your package:

{
    "name": "nova-mylib",
    "version": "1.0.0",
    "description": "A helpful utility library for Nova",
    "main": "mylib.nova",
    "author": "Your Name",
    "license": "MIT"
}

Step 3: Write Your Code

# mylib.nova - the main entry point

func capitalize(s) do
    import "string"
    if len(s) == 0 do
        return s
    done
    let first = string.to_upper(string.char_at(s, 0))
    let rest = string.substr(s, 1, len(s) - 1)
    return first + rest
done

func repeat(s, n) do
    let result = ""
    for i in range(0, n) do
        result = result + s
    done
    return result
done

The nova.json Manifest

The nova.json file describes your package. Here are all the supported fields:

"name" (required)
The package name. This is the identifier users will import. Should be lowercase, and may contain hyphens. Convention is to prefix with nova-.
"version" (required)
The semantic version of the package, e.g. "1.0.0", "0.3.2".
"description" (recommended)
A short description of what the package does.
"main" (required)
The entry point file, relative to the package root. This is the file that gets loaded when a user imports the package.
"author" (optional)
The name of the package author.
"license" (optional)
The license under which the package is distributed (e.g. "MIT", "Apache-2.0").
"build" (optional)
A shell command to run after the package is cloned. Used by native packages that contain C code and need to be compiled. The command runs inside the package directory.
"dependencies" (optional)
A dictionary of "name": "git-url" pairs. Transitive dependencies are installed automatically.

Full Example

{
    "name": "nova-http",
    "version": "0.2.1",
    "description": "HTTP client library for Nova",
    "main": "http.nova",
    "author": "Jane Doe",
    "license": "MIT"
}

Publishing to GitHub

To share your package with the Nova community, push it to a public GitHub repository.

Step 1: Initialize a Git Repository

cd nova-mylib
git init
git add .
git commit -m "Initial release"

Step 2: Create a GitHub Repository

Create a new repository on GitHub (e.g. username/nova-mylib), then push:

git remote add origin https://github.com/username/nova-mylib.git
git push -u origin main

Step 3: Tag a Release (Optional)

git tag v1.0.0
git push origin v1.0.0

That is it. Anyone can now install your package with:

nova install https://github.com/username/nova-mylib

Package Directory Structure

A well-organized package follows this layout:

nova-mylib/
  nova.json              # Package manifest
  mylib.nova             # Main entry point
  helpers.nova           # Additional modules
  mylib_test.nova        # Tests
  examples/
    basic.nova           # Usage examples
  README.md              # Documentation
  LICENSE                # License file

Tip: Include a README.md in your package with usage examples. It will display on your GitHub repository page.

Testing Your Package

Nova includes a built-in test runner. Create files ending in _test.nova and define test blocks:

# mylib_test.nova
import "mylib"

test "capitalize works on normal strings" do
    let result = mylib.capitalize("hello")
    assert(result == "Hello")
done

test "capitalize handles empty string" do
    let result = mylib.capitalize("")
    assert(result == "")
done

test "repeat creates repeated strings" do
    let result = mylib.repeat("ab", 3)
    assert(result == "ababab")
done

Run the tests from the command line:

nova test

The test runner will:

  1. Discover all files matching *_test.nova in the current directory
  2. Execute each test "name" do ... done block
  3. Report which tests passed and which failed
  4. Exit with code 0 if all tests pass, 1 otherwise

Assertions

Use the assert() function inside test blocks. If the expression is falsy, the test fails with an error message.

assert(1 + 1 == 2)                 # passes
assert(len([1, 2, 3]) == 3)       # passes
assert(type("hello") == "string")  # passes

CI Integration: Since nova test returns a non-zero exit code on failure, it integrates seamlessly with CI/CD systems like GitHub Actions.

Native Packages (C)

Nova packages can contain compiled C code in addition to .nova source files. This lets packages wrap C libraries (like raylib, SQLite, etc.) and expose them to Nova scripts.

A native package uses the "build" field in nova.json to compile its C code after installation. The build command should produce a shared library (.dylib on macOS, .so on Linux) in the package root directory.

Example: nova.json for a Native Package

{
    "name": "nova-game",
    "version": "1.0.0",
    "description": "2D/3D game development with raylib",
    "build": "cmake -B build && cmake --build build && cp build/game.dylib .",
    "author": "Cosmic",
    "license": "MIT"
}

When a user runs nova install, Nova will:

  1. Clone the repository into nova_packages/
  2. Detect the "build" field and run the command
  3. The compiled .dylib/.so is loaded automatically when the package is imported

Writing a Native Package

Your C code must export a single function called nova_module_init:

// game.c
#include "nova.h"

static NovaValue my_function(Interpreter *interp, int argc,
                             NovaValue *argv, int line) {
    return NOVA_INT(42);
}

void nova_module_init(Interpreter *interp, Environment *env) {
    nova_env_define(env, "my_function",
        NOVA_OBJ(nova_builtin_new("my_function", my_function, 0)));
}

Requirements: Users installing native packages must have the necessary build tools (CMake, a C compiler) and any system libraries the package depends on. Document these in your README.
