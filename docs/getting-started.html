Installation

Nova is implemented in C and builds with CMake. You will need a C compiler (gcc or clang) and CMake 3.10 or later.

Build from Source

# Clone the repository
git clone <repo-url>
cd nova

# Build with CMake
cmake -B build
cmake --build build

# Optionally install system-wide
sudo cmake --install build

Verify Installation

nova --version

You should see the version number printed. You are ready to write Nova programs.

Create a Project

Use nova init to scaffold a new project with a starter file and manifest:

nova init myapp
cd myapp
nova main.nova

This creates main.nova (starter program) and nova.json (project manifest).

Format Your Code

Nova includes a built-in formatter to keep your code consistent:

# Format a single file
nova fmt main.nova

# Format all .nova files in a directory
nova fmt src/

Hello World

Create a file called hello.nova with the following content:

print("Hello, World!")

Run it from your terminal:

nova hello.nova

That is it. No boilerplate, no main function, no imports. Nova runs your code top to bottom.

Tip: Nova files use the .nova extension by convention.

Variables and Types

Declare variables with the let keyword. Nova is dynamically typed, so you do not need to specify types.

# Variables
let name = "Nova"
let version = 1.0
let is_awesome = true
let nothing = none

Data Types

Nova has the following built-in types:

  • int — whole numbers: 42, -7, 0
  • float — decimal numbers: 3.14, -0.5
  • bool — boolean values: true, false
  • string — text: "hello" or 'hello'
  • list — ordered collection: [1, 2, 3]
  • dict — key-value map: {"key": "value"}
  • none — the absence of a value
  • function — first-class functions
  • class — class definitions
  • instance — class instances

Strings

Strings support escape sequences and expression interpolation with curly braces:

let name = "World"
print("Hello, {name}!")     # Hello, World!
print("2 + 2 = {2 + 2}")    # 2 + 2 = 4

# Escape sequences: \n \t \r \\ \' \"
print("Line 1\nLine 2")

Type Checking and Conversion

print(type(42))        # int
print(type("hello"))   # string

# Convert between types
let n = int("42")      # string to int
let s = str(42)        # int to string
let f = float("3.14")  # string to float

Truthiness

The following values are considered falsy: none, false, 0, "" (empty string), and [] (empty list). Everything else is truthy.

Control Flow

If / Else

let age = 18

if age >= 18 do
    print("You can vote!")
else do
    print("Too young to vote.")
done

While Loops

let count = 0
while count < 5 do
    print(count)
    count = count + 1
done

For-In Loops

# Iterate over a list
let fruits = ["apple", "banana", "cherry"]
for fruit in fruits do
    print(fruit)
done

# Iterate over a range
for i in range(1, 6) do
    print("Count: {i}")
done

Break and Continue

for i in range(1, 100) do
    if i % 2 == 0 do
        continue          # skip even numbers
    done
    if i > 10 do
        break             # stop after 10
    done
    print(i)
done

Logical Operators

if age >= 18 and has_id do
    print("Access granted")
done

if not is_banned or is_admin do
    print("Welcome")
done

Functions and Lambdas

Defining Functions

func greet(name) do
    return "Hello, {name}!"
done

print(greet("Nova"))  # Hello, Nova!

Multiple Parameters

func add(a, b) do
    return a + b
done

print(add(3, 4))  # 7

First-Class Functions

Functions are values. You can store them in variables and pass them to other functions.

func apply(f, value) do
    return f(value)
done

func double(x) do
    return x * 2
done

print(apply(double, 5))  # 10

Lambdas (Arrow Functions)

let square = (x) => x * x
print(square(5))  # 25

let add = (a, b) => a + b
print(add(3, 4))  # 7

Ternary Expressions

let label = "even" if n % 2 == 0 else "odd"

The Pipe Operator

The pipe operator |> passes the result of the left expression as the first argument to the right function:

let result = "  hello world  "
    |> string.trim()
    |> string.to_upper()

print(result)  # HELLO WORLD

Classes

Nova supports object-oriented programming with classes, constructors, methods, and single inheritance.

Defining a Class

class Animal do
    func init(name, sound) do
        self.name = name
        self.sound = sound
    done

    func speak() do
        print("{self.name} says {self.sound}!")
    done
done

let dog = Animal("Rex", "Woof")
dog.speak()  # Rex says Woof!

Inheritance

class Dog(Animal) do
    func init(name) do
        self.name = name
        self.sound = "Woof"
    done

    func fetch(item) do
        print("{self.name} fetches the {item}!")
    done
done

let buddy = Dog("Buddy")
buddy.speak()           # Buddy says Woof!
buddy.fetch("ball")     # Buddy fetches the ball!

Note: Use self to access instance properties and methods. The init method is called automatically when you create a new instance.

Working with Collections

Lists

let fruits = ["apple", "banana", "cherry"]

# Access by index
print(fruits[0])         # apple

# Length
print(len(fruits))       # 3

# Add elements
append(fruits, "date")

# Iterate
for fruit in fruits do
    print(fruit)
done

Dictionaries

let person = {
    "name": "Alice",
    "age": 30,
    "city": "Wonderland"
}

# Access values
print(person["name"])    # Alice

# Get all keys and values
let k = keys(person)     # ["name", "age", "city"]
let v = values(person)   # ["Alice", 30, "Wonderland"]

# Add or update entries
person["email"] = "alice@example.com"

Modules and Imports

Use import to load other Nova files or standard library modules.

Each imported module becomes a namespace — access its functions and constants with dot notation.

Standard Library Imports

import "math"
import "io"
import "string"

print(math.PI)               # 3.14159...
print(math.sqrt(16))         # 4
print(string.to_upper("hi")) # HI

Importing Your Own Files

# utils.nova
func greet(name) do
    return "Hello, {name}!"
done
# main.nova
import "utils"

print(utils.greet("World"))

C Shared Libraries

Nova can also load C shared libraries as modules, enabling native extensions for performance-critical code.

Error Handling

Use try/catch to handle runtime errors gracefully.

try do
    let result = 10 / 0
catch err do
    print("Error: {err}")
done

You can use try/catch with file I/O, network operations, type conversions, and any other code that might fail at runtime:

import "io"
import "json"

func load_config(path) do
    try do
        let data = io.read_file(path)
        return json.parse(data)
    catch err do
        print("Failed to load config: {err}")
        return {}
    done
done

let config = load_config("settings.json")
